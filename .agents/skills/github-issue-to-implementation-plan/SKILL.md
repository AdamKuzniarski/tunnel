---
name: github-issue-to-implementation-plan
description: Turn a GitHub issue into a scoped implementation plan before editing code. Use for issue-driven development, small patches, and branch-per-ticket workflow.
---

Before editing code:
1. Read the issue goal, tasks, done-when, and Codex rules.
2. Find the likely files.
3. Explain the current state.
4. Propose the smallest safe implementation plan.
5. Do not code until the plan is clear.
6. Respect existing business logic unless the issue explicitly asks to change it.
