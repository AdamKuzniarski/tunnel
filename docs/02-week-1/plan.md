# Week 1 Plan

Week 1 is a feasibility spike.

The goal is not to finish the product.
The goal is to reduce uncertainty.

## Main week-1 goals

- create a clean repository base
- define the product direction
- define the MVP
- describe the technical architecture
- initialize the mobile app structure later in the week
- understand the main iOS-native feasibility risks

## Questions this week should answer

- Is the repository structure clear enough to support the project?
- Is the product scope realistic?
- Are the architecture boundaries understandable?
- What are the biggest iOS-specific risks?
- What must be proven before going deeper?
